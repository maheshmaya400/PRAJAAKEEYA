# 🔐 Security Policy

## Scope

**PRAJAAKEEYA** is a documentation and educational content repository. It does not currently
host executable application code, servers, or user data collection systems. However, we take
the integrity of this repository — and any future tooling built around it — seriously.

---

## Supported Versions

| Version | Supported |
|---|---|
| `main` branch | ✅ |
| Older tagged releases | ⚠️ Best effort only |

---

## Reporting a Vulnerability

If you discover a security issue — including but not limited to:

- Malicious links or content injected via a pull request
- Compromised assets (images, scripts) that could affect users
- Any future web/app component vulnerabilities (XSS, data exposure, etc.)

Please **do not** open a public issue. Instead:

1. Report privately via the contact channels listed in [README.md](README.md)
   (e.g., a dedicated security email or maintainer DM).
2. Include a clear description, steps to reproduce, and potential impact.
3. Allow maintainers a reasonable window to investigate and patch before public disclosure.

## Content Integrity

Since this is a politically sensitive educational topic, we also treat **misinformation,
defamatory content, or politically biased insertions** as a "content security" issue. Please
report such content the same way you would report a technical vulnerability.

---

## Our Commitment

- We will acknowledge reports within a reasonable timeframe
- We will investigate and, where applicable, patch or remove harmful content promptly
- We will credit responsible reporters (if desired) once an issue is resolved

Thank you for helping keep this project safe, accurate, and trustworthy.
